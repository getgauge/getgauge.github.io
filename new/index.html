Why Gauge is better for Testers
Testers
Developers

  • Simple Syntax

    Write specifications in Markdown, a widely known markup language. Gauge won't enforce a structure: write in a way that works for you. Generate readable documentation in the format of your choice.

  • Get Started Fast

    Gauge is lightweight and needs no configuration needed. A single command plugin install and initialisation. And you’re good to go.

  • Your Environment, Your Tools

    Gauge supports C#, Java, Ruby, IntelliJ and Visual Studio. Automate in your favourite programming language and work in the IDE of your choice, across platforms.

  • Data Driven Execution

    Easily test with large data sets, while keeping specifications highly readable. Gauge reads test data from text, CSV, and other sources.

  • Refactoring support

    Test suites need to be as DRY and as malleable as your production code. Gauge's robust refactoring propagates changes throughout the test suite, reducing duplication and evolving your tests with your application.

    Run your tests in parallel with a single command from the IDE or command line. It'll work exactly the same across all languages.

Why did we build Gauge?

  • We are passionate about open source and test automation.
  • We want to be the tool of choice for testers regardless of their automation experience.
  • Building a community of people with varying preferences is important to us. And Gauge is built with a plugin supportive architecture that will nurture this vision and adapt to accommodate it.
  • ThoughtWorks, creator of tools like Selenium and Twist, has been a key player in the automated testing ecosystem for more than a decade. We love it. We're here to stay and we want to get even better to solve the hardest problems.
